1.
a) Live Data - класс хранителя данных, что можно наблюдать. Хранит последнюю версию данных
b) ViewModel - предоставляет данные для ПИ. Действует как центр связи между репохиторием и ПИ
c) Dao - объект доступа к данным. Отображение запросов SQL на функции
d) Repository - класс, для управления источниками данных
e) Room - держатель БД, что служит точкой доступа к ней, использует ДАО для запросов
2. Lifecycle - объект, на который можно повесить слушателей, что будут получать уведомления при изменении состояния активити
3. помещение в него и получение из него объектов
4. создать класс, наслед от AndroidViewModel, ViewModelProviders для связи с активити
5. с помощью getData и loadData

Room - держатель БД, что служит точкой доступа к ней, использует ДАО для запросов
Dao - объект доступа к данным. Отображение запросов SQL на функции

7. DataBinding позволяет организовывать работу с Вью таким образом, чтобы не нужно было писать много методов. Для привязки включить в билд градл
8.Observable позволяет обновлять элементы ПИ при изменении наблюдаемого значения
